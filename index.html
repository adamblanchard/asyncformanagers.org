  1. Transparent and public communication over information control.
  2. Written information over verbal meetings.
  3. Flexible working hours over synchronised schedules.
  4. Remote-working over co-location.
  5. Trust and ownership over task delegation.

Meetings as a last resort

Meetings are an easy option but realise that they are really expensive. Whenever you need to talk to someone, always ask yourself how you can do it in an asynchronous way first. Once those options are exhausted, a meeting can be your last option.

Sync time is limited, and it's not just yours to use up, so use it appropriately

Sync time or meetings are not evil but you do have a very limited amount of it available in a day. If you ask someone how many hours they would genuinely choose to be in meetings each day, it would likely be a very small number. Remember, when you invite someone to talk you are taking away their precious uninterupted focus time.

Instead, focus on finding the right time to make use of it. If you sense some conflict or misunderstanding, need to build some human connection with people (useful when working remotely), or otherwise need to collaborate in real time, go for it.

Overcommunicate in public by default

Overcommunicating is valuable because in async you can't (and shouldn't) rely on instant replies, so giving more context and considering what someone might ask straight away means you can proactively squish follow ups being necessary.

Here's what you should include:
  1. Enough details to cover follow up questions - don't let it turn into a long cycle conversation.
  2. Response time expectations - when do you need a reply by?
  3. As much context and supporting info as possible - documentation, tickets, screenshots, all that good stuff.
  4. A clear need and expectation - are you looking for approval? Feedback? A decision? Let them know.

Documentation is scalable

Writing something down publicly builds scalable knowledge. You don't have to keep having the same meetings to relay the same info to different people. If new people join you don't have to remember all the important things and talk it through to each person. People can proactively find the info they need themselves. If you leave the team there is still some context around.

Go synchronous when it gets synchronous

If you end up having a real time discussion in slack or email, realise that you might be able to communicate more effectively on a call. If you do this, remember to follow up with a written summary so those who missed it can catch up asynchronously.

Lead by example

As a manager, the most effective way you can influence a successful asynchronous culture is to live by the values yourself. Don't ask a team to send public and async updates if you yourself call them into a meeting every time you need to tell them something.
